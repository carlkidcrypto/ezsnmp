=====================================
Fedora 41 with net-snmp 5.7 Docker
=====================================

Overview
========

This Docker image provides a testing environment based on Fedora 41 (latest stable) with:

* **Base OS**: Fedora 41
* **GCC**: Native GCC 14.x (g++ 14.x, exceeds g++ 9.5 minimum — no toolset wrapper needed)
* **Python Versions**: 3.10, 3.11, 3.12, 3.13, 3.14 (all from source builds)
* **Virtual Environment**: /opt/venv (Python 3.14 with all project dependencies pre-installed)
* **net-snmp**: 5.7.3 (built from source against OpenSSL 1.0.2u)
* **OpenSSL**: 1.0.2u (built from source, installed to /usr/local/openssl10)
* **C++ Testing**: Full g++ 14 support for cpp_tests with coverage reporting
* **Python Testing**: Full support for python_tests with coverage reporting
* **Image Size**: Optimized to minimal size through consolidated RUN statements and cache cleanup

Building
========

To build the image locally:

.. code-block:: bash

    cd docker/fedora41_netsnmp_5.7
    docker-compose build

Running
=======

To run the container with automatic setup:

.. code-block:: bash

    cd docker/fedora41_netsnmp_5.7
    ./go_docker.sh

Or manually:

.. code-block:: bash

    docker-compose up -d
    docker exec -it fedora41_netsnmp_5.7_snmp_container /bin/bash

Testing
=======

Inside the container, run tests with:

.. code-block:: bash

    # Python tests
    cd /ezsnmp
    pytest python_tests/

    # C++ tests
    cd /ezsnmp/cpp_tests
    ./get_test_coverage.sh

GCC
===

Fedora 41 ships with native GCC 14.x which provides:

* GCC 14.x compiler suite (no toolset wrapper required)
* Support for modern C++ features
* Full compatibility with C++17/C++20/C++23 standards

The compiler is available system-wide without any PATH modifications.

net-snmp 5.7.3
==============

net-snmp 5.7.3 requires OpenSSL 1.0.x — it is not compatible with the system
OpenSSL 3.x shipped by Fedora 41. OpenSSL 1.0.2u is therefore built from source
and installed into ``/usr/local/openssl10``. net-snmp is then configured to use
that private OpenSSL installation via ``--with-openssl=/usr/local/openssl10``.

The build uses ``-j1`` and ``MALLOC_CHECK_=0`` for stability. The
``ENV MALLOC_CHECK_=0`` and ``ENV LD_LIBRARY_PATH`` environment variables are set
in the image to ensure the OpenSSL 1.0 shared libraries are found at runtime.
